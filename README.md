## 📝 项目简介

本项目通过 Coze 可视化工作流，实现了从“输入人名”到“输出剪映草稿”的全自动处理。工作流严格遵循历史叙事逻辑，并内置了**内容安全审查机制**，确保所有生成的画面符合“幼儿园可播放标准”（即无暴力、血腥或敏感内容，全部转化为教学、演练或舞台剧形式）。

## ✨ 核心特性

*   **全自动化流程**：一键输入人物名称，自动生成文案、配图、视频片段及字幕。
*   **历史叙事优化**：LLM 专注于提取人物生平的 6 个关键节点，确保故事线紧凑。
*   **安全合规**：独特的“安全提示词”转换机制，将战争、冲突等场景转化为“历史研究中心模拟”、“舞台剧排练”等安全场景。
*   **剪映无缝对接**：利用剪映小助手插件，自动生成包含视频、音频、字幕的剪映草稿（Draft），方便后期微调。
*   **中国风视觉**：默认采用“中国古代写实风格”，支持超高清晰度与细腻质感渲染。

## 🛠️ 技术栈与依赖

*   **编排平台**：[Coze (扣子)](https://www.coze.cn/)
*   **核心模型**：
    *   **文案生成**：LLM (支持 JSON 格式输出)

    *   `timelines` / `video_infos` / `caption_infos`
    *   `create_draft` / `add_videos` / `add_captions`

## ⚙️ 工作流程详解

整体流程分为四个主要阶段：**文案构建**、**素材生成**、**安全转化**、**视频合成**。

### 第一阶段：文案构建 (LLM)
1.  **输入**：用户输入历史人物名称（如“拿破仑”、“朱元璋”）。
2.  **生成内容**：LLM 根据系统提示词，输出包含 `title`（标题）和 `content`（生平6大节点）的 JSON 数据。
    *   *限制*：仅限6个关键事迹，编年体叙述。

### 第二阶段：视觉素材生成 (Image & Video)
3.  **图片提示词**：LLM 将文案转化为具体的绘画描述词（如“35岁的朱元璋身穿华贵甲胄...”）。
4.  **图像生成**：调用文生图模型，按比例 9:16 生成对应历史场景图片。
5.  **视频提示词**：再次调用 LLM，结合**安全准则**，将图片提示词转化为安全的视频生成提示词（如将“战场”转为“沙盘推演”）。
6.  **视频生成**：使用图文转视频功能，以生成的图片为首帧，生成动态视频片段。

### 第三阶段：数据处理与格式化
7.  **链接提取**：通过 `get_url` 和 `objs_to_str_list` 提取有效的视频 URL。
8.  **时间线规划**：根据视频数量和预估时长，计算 `timelines`（时间线列表）。
9.  **资源准备**：准备背景音乐 URL，并生成音频时间线。

### 第四阶段：剪映草稿合成
10. **创建草稿**：初始化一个 1080x1920 (竖屏) 的剪映草稿。
11. **批量添加**：
    *   添加视频片段 (`add_videos`)
    *   添加字幕 (`add_captions`，支持自定义字体如“江湖体”、颜色、位置)
    *   添加标题 (`add_captions`，独立样式)
    *   添加背景音乐 (`add_audios`)

## 🚀 如何运行 (Setup Guide)

由于这是一个 Coze 工作流配置，您不能直接运行代码，需要按照以下步骤在 Coze 平台上复现：

1.  **创建工作流**：登录 Coze，新建一个工作流。
2.  **导入节点**：参照下方流程图，依次添加以下节点：
    *   **开始节点**：定义变量 `name`。
    *   **LLM 节点 x3**：分别配置“生成文案”、“生成图片提示词”、“生成视频提示词”（注意复制文档中的系统提示词）。
    *   **批处理节点**：用于循环处理数组数据（图片生成、视频生成）。
    *   **代码节点**：配置 `图片格式转换` 和 `预估视频时间`。
    *   **插件节点**：添加所有以 `剪映小助手:` 开头的插件（需先在插件市场安装）。
3.  **连线**：严格按照逻辑顺序连接各节点输入输出。
4.  **测试**：在试运行界面输入“诸葛亮”或“秦始皇”，观察是否成功生成 `draft_id`。

## ⚠️ 注意事项

1.  **安全限制**：视频生成提示词节点强制要求“幼儿园可播放标准”，如需生成写实战争片，需修改该节点的系统提示词。
2.  **配额限制**：图像生成和视频生成均为付费或消耗积分操作，请注意账户余额。
3.  **剪映版本**：需确保本地安装了剪映专业版，且剪映小助手插件版本兼容。
4.  **备用图片**：在图片格式转换节点中，建议配置 `beiyong` (备用) 参数，以防部分图片生成失败导致视频断层。
